# Workflow Baseline

## Current IDE Setup

### IDE and Tools Used

- Visual Studio Code
- GitHub Copilot
- GitHub Copilot Chat
- Prettier
- ESLint
- Python Extension
- C/C++ Extension
- Git and GitHub integration

### Programming Languages

- Python
- JavaScript
- C

### Development Workflow

1. Write and edit code in VS Code.
2. Use GitHub Copilot for code suggestions and debugging assistance.
3. Run local tests using terminal commands.
4. Commit changes to GitHub regularly.
5. Review and fix issues manually after testing.

---

## Pain Points

### Manual Debugging Time

Debugging runtime and logical errors manually can take a significant amount of time, especially when reviewing large files or complex logic.

### Repetitive Documentation Tasks

Creating README files, API documentation, and structured reports manually is repetitive and slows down development.

### Context Switching

Frequently switching between coding, testing, debugging, and documentation interrupts workflow concentration and reduces efficiency.

### Test Case Creation

Writing edge-case tests and repetitive unit tests manually requires additional development time.

---

## Productivity Metrics

### Average Task Completion Time

- Simple coding task: 20–30 minutes
- Medium complexity task: 45–90 minutes
- Documentation task: 30–60 minutes

### Bug Fix Turnaround

- Minor bug fixes: 10–20 minutes
- Moderate debugging tasks: 30–60 minutes
- Complex debugging sessions: 1–2 hours

### Commit Frequency

- Average commits per week: 20–30 commits

### Documentation Productivity

- Average API documentation creation time: 45 minutes
- Average report writing time: 30 minutes

---

## Current Workflow Challenges

- Maintaining consistent coding standards across files
- Repeating similar debugging patterns manually
- Managing project documentation updates
- Balancing development speed with testing quality

---

## Baseline Observations

The current workflow is functional but contains repetitive manual tasks that reduce productivity. AI-assisted development tools such as GitHub Copilot help accelerate coding, debugging, and documentation generation. However, manual testing and validation remain necessary to ensure code quality, correctness, and project compliance.
