# Prompt Use Cases

## Code Quality

### Refactoring

- **Goal:** Improve readability and maintainability of existing code.
- **Input:** Source code in Python, JavaScript, or C.
- **Expected Output:** Cleaner and optimized code with explanations.

### Style Enforcement

- **Goal:** Apply consistent naming conventions and formatting.
- **Input:** Poorly formatted source code.
- **Expected Output:** Reformatted code following coding standards.

### Performance Optimization

- **Goal:** Reduce execution time and improve efficiency.
- **Input:** Slow or repetitive code logic.
- **Expected Output:** Optimized implementation with performance notes.


---

## Debugging

### Syntax Error Detection

- **Goal:** Identify invalid syntax preventing execution.
- **Input:** Source code with parsing errors.
- **Expected Output:** Corrected syntax with explanation of the issue.

### Runtime Exception Analysis

- **Goal:** Diagnose runtime crashes and exceptions.
- **Input:** Code and error messages.
- **Expected Output:** Root cause analysis and corrected code.

### Logical Error Identification

- **Goal:** Find incorrect program behavior despite successful execution.
- **Input:** Program output and source code.
- **Expected Output:** Explanation of faulty logic and corrected implementation.


---

## Documentation

### Code Documentation

- **Goal:** Generate comments and explanations for source code.
- **Input:** Existing code without documentation.
- **Expected Output:** Well-commented code and function descriptions.

### API Documentation

- **Goal:** Explain API endpoints and usage.
- **Input:** Backend API source code.
- **Expected Output:** Structured API documentation with request and response examples.

### README Generation

- **Goal:** Create project overview documentation.
- **Input:** Project structure and features.
- **Expected Output:** Complete README file with setup instructions.


---

## Testing

### Unit Test Generation

- **Goal:** Create automated tests for functions.
- **Input:** Source code functions.
- **Expected Output:** Unit test cases with expected assertions.

### Edge Case Testing

- **Goal:** Identify unusual or extreme input scenarios.
- **Input:** Program requirements and source code.
- **Expected Output:** Additional edge case test scenarios.

### Bug Reproduction

- **Goal:** Reproduce reported issues consistently.
- **Input:** Error descriptions and affected code.
- **Expected Output:** Steps and test cases to reproduce the bug.


---

## Security

### Vulnerability Detection

- **Goal:** Identify insecure coding practices.
- **Input:** Application source code.
- **Expected Output:** Security analysis and safer alternatives.

### Input Validation Review

- **Goal:** Detect missing or weak input validation.
- **Input:** User input handling code.
- **Expected Output:** Validation improvements and recommendations.

### Authentication Review

- **Goal:** Verify authentication and authorization logic.
- **Input:** Login and session management code.
- **Expected Output:** Security findings and corrections.


---

## Automation

### Script Generation

- **Goal:** Automate repetitive development tasks.
- **Input:** Task description and workflow requirements.
- **Expected Output:** Automation scripts or command sequences.

### Deployment Assistance

- **Goal:** Simplify application deployment processes.
- **Input:** Deployment environment details.
- **Expected Output:** Deployment scripts and configuration guidance.

### Log Analysis

- **Goal:** Analyze logs to identify failures or anomalies.
- **Input:** Application log files or console outputs.
- **Expected Output:** Summary of detected issues and troubleshooting suggestions.
